## 📖 About Mira

Mira acts as your intelligent partner for understanding software code. Projects often contain thousands of files. Finding specific logic or connecting ideas across these files takes time. Mira reads through your codebase and organizes the information into clear summaries. You use these summaries to ask questions, solve bugs, or plan new features. It removes the guesswork from development by providing deep insights into how your files interact.

## 🛠 Features

*   **Codebase Mapping**: The tool builds a connection graph of all files in your folder. It identifies which parts of the code talk to other parts.
*   **Context Engineering**: Mira gathers relevant snippets from your project to answer your specific technical questions accurately.
*   **Signal Digests**: You receive structured reports that highlight recent changes and important updates within your repository.
*   **Context Packs**: You save and load specific project states so your work stays organized across sessions.
*   **Offline Operation**: Mira processes your documents locally. Your private code stays on your machine during the analysis phase.

## ⚙️ Understanding the Interface

The Mira interface consists of three main parts. The Sidebar shows your project file tree and saved context packs. The Central Panel displays the conversation with the software. The Settings Menu at the top right allows you to adjust how much depth the tool uses during each search. Keeping the depth setting on Automatic works well for most medium projects. If your project exceeds one thousand files, use the Comprehensive setting to ensure full coverage.

## 🧪 Advanced Usage Tips

*   **Filtering**: You can tell Mira to ignore certain folders, like node_modules, to speed up the analysis phase. Use the settings tab to create an ignore list.
*   **Updating**: Mira checks for updates every time you open the program. Install these updates to improve the accuracy of the code analysis.
*   **Saving Progress**: When you finish a session, save your findings as a context pack. This allows you to pick up exactly where you exited later.
*   **Querying**: Use short sentences. Instead of pasting a massive block of code, ask Mira to explain the intent behind a specific function name.

## 🛡 Security and Privacy

Mira functions locally on your machine. The software does not send your raw source code to external servers. Metadata about the project structure is processed through the analysis engine to generate answers, but your sensitive variables and credentials remain untouched. The program uses specific security protocols to ensure that all internal file access is restricted to the folders you explicitly allow in the application settings.

## ❓ Frequently Asked Questions

**Does Mira modify my source code?**
No. Mira acts as a read-only tool. It scans your files to learn, but it never writes or changes any of your original files.

**How much time does the initial scan take?**
For a small project, the process takes seconds. Large projects with many files may take a minute or two to organize.

**Can I run Mira on older versions of Windows?**
Mira requires recent library support found in Windows 10 and 11. Older versions of Windows may lack the necessary components to run the installer successfully.

**What happens if I lose my internet connection?**
Mira continues to work without an internet connection for most local analysis tasks. However, updates or specific cloud-enhanced features require an active connection to function.

**Does this require programming knowledge?**
No. While Mira is built by and for developers, you do not need to write code to use it. You only need to provide the path to your code folder and read the plain English reports it generates.

**How do I adjust the tool if the results seem off?**
Check your settings to ensure the correct folders are included. If the tool struggles with a specific file type, verify that the file extension is supported in the general preferences menu.

**Where does the data go?**
All data stays on your local machine. No information is stored on external cloud storage unless you manually export a report to a file location of your choice.
